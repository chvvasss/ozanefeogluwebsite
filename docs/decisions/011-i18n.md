# ADR-011 — i18n: Path-based Locale Routing + Spatie Translatable JSON Columns

## Status
Accepted — 2026-04-18

## Context
Brief Bölüm 4.4: TR + EN, içerik seviyesinde çeviri, hreflang doğru. Kararlar:
- **Locale tespiti:** URL path mı? Subdomain mı? Cookie/header mı?
- **İçerik çevirisi:** Ayrı kayıt mı, JSON kolon mu, ayrı tablo mu?
- **UI string'leri:** `lang/` translation files.
- **SEO:** hreflang, canonical.

### Hedefler
- TR default + canonical (sahibi Türkiye, ana hedef kitle TR).
- EN ayrı path: `/en/...`.
- Her içerik öğesi (post, project, page, category, tag, menu item) per-locale slug + per-locale alanlar.
- Bazı içerikler tek dilde olabilir (örn: bir blog yazısı yalnız TR yayınlanır); UI bunu graceful handle eder ("EN versiyonu yok").

## Decision

### Routing
**Path-based locale prefix.** Default locale (TR) prefix-siz. Diğer locale'ler `/en/...`.

```php
// routes/web.php
Route::group(['prefix' => '{locale?}', 'where' => ['locale' => '(en)?']], function () {
    Route::get('/', [HomeController::class, 'index']);
    Route::get('/works', [WorkController::class, 'index']);
    // ...
});
```

Middleware `SetAppLocale`:
1. `route()->parameter('locale')` varsa → set.
2. Yoksa → default (`tr`).
3. Set `App::setLocale($locale)`.

### İçerik çevirisi
**Spatie Translatable** — JSON kolon, locale-keyed.

```php
// schema
$table->json('title');         // {"tr": "Başlık", "en": "Title"}
$table->json('body');
$table->json('excerpt');
$table->json('meta_title');
$table->json('meta_description');
// slug ayrı (per-locale unique)
$table->json('slug');          // {"tr": "tr-slug", "en": "en-slug"}
```

Model:
```php
use Spatie\Translatable\HasTranslations;

class Post extends Model {
    use HasTranslations;
    public $translatable = ['title', 'body', 'excerpt', 'meta_title', 'meta_description', 'slug'];
}
```

Erişim:
- `$post->title` → app locale değer (auto).
- `$post->getTranslation('title', 'en')` → spesifik locale.
- `$post->getTranslations('title')` → tüm map.

### Slug uniqueness
- Per-locale unique check: `where("slug->{$locale}", $value)` (MySQL JSON path).
- Helper: `Post::whereTranslation('slug', $value, $locale)`.

### URL routing — slug to model
```php
Route::get('/writing/{slug}', [PostController::class, 'show'])->name('post.show');

// Controller
public function show(string $slug, string $locale = 'tr')
{
    $post = Post::published()
        ->whereTranslation('slug', $slug, $locale)
        ->firstOrFail();
    return view('post.show', ['post' => $post]);
}
```

### UI string'leri
- `lang/tr/`, `lang/en/` directories. Klasik Laravel.
- Kullanım: `__('messages.welcome')`, `@lang('messages.welcome')`.

### Hreflang
Her sayfa head'inde:
```html
<link rel="alternate" hreflang="tr" href="https://ozanefeoglu.com/writing/sade-yazilim">
<link rel="alternate" hreflang="en" href="https://ozanefeoglu.com/en/writing/on-simple-software">
<link rel="alternate" hreflang="x-default" href="https://ozanefeoglu.com/writing/sade-yazilim">
<link rel="canonical" href="https://ozanefeoglu.com/writing/sade-yazilim">
```

Translation eksikse o locale link rendered edilmez.

### Eksik çeviri davranışı
- **Locale değiştirici tıklanır + çeviri yok:** "Bu içerik henüz İngilizce'ye çevrilmedi" notu + ilgili locale anasayfa link.
- **Eksik UI string:** Fallback locale (TR) string. Logger'a `LocaleHasFallbacked` event (development'ta görünür).

### Admin UI
- Form'da her translatable field için locale tab'ları: `[TR] [EN]`.
- Active tab → ilgili locale field'ı görünür.
- Slug otomatik (her locale için title'dan), manuel override ile.
- "Translate from TR (auto)" butonu (gelecekte Deepl/Google API entegre edilebilir; şimdilik manuel).

## Consequences

### Pozitif
- **JSON kolon yaklaşımı:** Ayrı translation tablosuna göre 1 join az; query basit.
- **Spatie Translatable olgun:** 6+ yıl, tested.
- **SEO doğru:** hreflang, locale-aware canonical.
- **Path-based:** Subdomain'e göre daha basit (cert, DNS, hosting tek).
- **Default locale prefix-siz:** TR kanonik URL'ler temiz: `/writing/sade-yazilim` (`/tr/writing/...` değil).

### Negatif / Trade-off
- **JSON kolon FULLTEXT zorlaşır:** MySQL JSON path FULLTEXT index sınırlı. Mitigation: search Meilisearch (ADR-008), DB FTS zaten ikincil.
- **Migration:** Bir alan translatable yapmak için existing data migration. Mitigation: helper command + dökümante edildi.
- **Slug uniqueness fonksiyonel index:** MySQL 8'de `ALTER TABLE posts ADD INDEX ((CAST(slug->>"$.tr" AS CHAR(255))) ASC)` — kabul edilebilir karmaşıklık.

### Risk
- **EN çeviri hiç yapılmaz:** Site yarı boş hisseder. Mitigation: opsiyonel "EN sürümü taslak" görünmez. Sahibi karar verir; ayar.
- **Çoklu locale slug çakışması:** TR `iletisim` ile EN `contact` ayrı; ama bir gün üçüncü locale eklenir + slug çakışırsa uniqueness check kapsamı genişler. Test coverage.

## Alternatives Considered

### Subdomain (`tr.ozanefeoglu.com`, `en.ozanefeoglu.com`)
- **Pro:** Locale tamamen separate.
- **Con:** SSL cert + DNS karmaşık. SEO genelde aynı domain'de yapılır; subdomain Google için "ayrı site" sinyali (kötü).
- **Karar:** Reddedildi.

### Domain bazlı (`.com.tr` / `.com`)
- **Pro:** Yerel SEO sinyali.
- **Con:** İki ayrı site sayılır; geri-link/auth/içerik yönetimi karmaşık.
- **Karar:** Reddedildi.

### Ayrı translation tablosu
- **Pro:** Normalized, FULLTEXT kolay.
- **Con:** Her query'de join; UI form karmaşık.
- **Karar:** Reddedildi (Spatie Translatable JSON pratik).

### Cookie/header bazlı locale (URL'de yok)
- **Pro:** Tek URL, locale arka planda.
- **Con:** SEO felaket — Google iki locale'i farklı URL'de göremez.
- **Karar:** Reddedildi.

### Tek dil (TR only) ilk lansman, EN sonra
- **Pro:** Kapsam küçük.
- **Con:** Mimariye sonradan i18n eklemek migration ödevi büyük. Brief planlı i18n.
- **Karar:** Reddedildi (ilk günden i18n-ready).

## Locale ekleme prosedürü (gelecekte)
1. `config/app.php` `available_locales` array'ine ekle (örn. `'de'`).
2. Translatable model'lerde mevcut data otomatik fallback'e düşer.
3. `lang/de/` directory oluştur.
4. Admin form'larında otomatik yeni locale tab'ı görünür.
5. Routing param regex genişle (`(en|de)?`).
6. hreflang otomatik dahil eder.
